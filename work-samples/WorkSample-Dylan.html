Data Engineering¶

It's important to perform text mining on the tweets to ensure that emojis and numeric values are not included as input into the prediction model. If input data is not of good quality, then garbage into a model = garbage out¶

When text mining, we could perform lemmatization that "groups together" inflections of the same word - this can come from stemming words be taking "admirals" to be "admir". We could use lemmatization when identifying matches between words, but it's used on a case-by-case basis. Unless it improves model out-of-sample performance, we don't need to use PorterStemmer, for instance.¶

To solve the issue of chains of words in hashtags, we can use dynamic programming. We can split a chain of words in many ways such as "Imanage" becoming "I man age" or "I manage". Not all splits will result in a valid set of words. And not all valid splits are equally likely. We want the most likely split in the chain of words.¶

In [16]:
from nltk.corpus import words
# add 1 to avoid log of 0 error
wordList = words.words()
numWords = len(wordList)
costs = {word:(np.log((rank+1)*(np.log(numWords)))) for rank, word in enumerate(wordList)}
longestWordLength = max(list(map(lambda word: len(word), wordList)))

# A naive approach would not rely on memoization and use recursion to obtain prefixes along with the costs, 
# and reducing to find the sequence of words resulting in the minimum cost, greatest likelihood of
# occurring.
# We could store a memoization table:
def splitWordChain(chain: str) -> list:
    dim = len(chain)
    memo = [(0,0)]*(dim+1)
    splitChain = []
    for end in range(1,dim+1):
        minCostAndWord = (float('inf'),0)
        for start in range(max(0,end-longestWordLength), end):
            costAndWord = (memo[start][0]+costs.get(chain[start:end],float('inf')), end-start)
            if costAndWord < minCostAndWord: 
                minCostAndWord = costAndWord
        memo[end] = minCostAndWord
    # one could merge the two loops together and instead have 1 loop, but here 
    # both are in same big-O.
    pos = dim
    # in reverse to obtain min-cost backwards - first in memo costs is a placeholder - 0's.
    while pos > 0:
        cost, length = memo[pos]
        if length == 0:
            return '' # no valid splitting
        splitChain += [chain[pos-length:pos]]
        pos -= length
    return ' '.join(reversed(splitChain))

from nltk.stem import PorterStemmer
ps = PorterStemmer()
def mineTweet(text):
    # set that doesn't necessarily contribute to the overall sentiment
    airlines = {"virginamerica","usairways",
                "americanair","united",
                "southwestair","jetblue"}
    singleLetters = {'a','u','y','r'}
    # all lowercase
    OutText = text.lower()
    OutText = re.sub(r'<[^<]+?>','', OutText)
    
    # removing unimportant links - http - can affect length of sentence - 
    # "document size/length" and remove any non-space values after http
#     OutText = re.sub(r'http\S+',' ', OutText)
    OutText = ' '.join([splitWordChain(item[1:]) if (item.startswith("#") and item[1:].isalpha()) else item 
                        for item in OutText.split()])
    OutText = ' '.join([item for item in OutText.split() if "http" not in item])
    # remove emojis - after demojizing, the text will have 
    # colons : around the text form of the emoji.
    OutText = re.sub(r':', ' ', emoji.demojize(OutText))
    # we remove symbols and numbers from the text
    OutText = re.sub(r'[^a-z]', ' ', OutText)
    
    # we split the sentence into individual words
    words = OutText.split()
    # we remove words that are from airlines
    words = [ps.stem(word) for word in words if (word not in airlines) and (len(word) > 1 or (len(word)==1) and word in singleLetters)]
    return words

def labelVader(text):
    compoundScore = vaderModel1.polarity_scores(text)['compound']
    if compoundScore == 0:
        return "neutral"
    elif compoundScore > 0:
        return "positive"
    else:
        return "negative"

Method 3: Random Forests and Naive Bayes¶

With regards to choosing the right machine learning model to predict sentiment in text, we have looked at the previous two models that were pretrained and have mediocre accuracy. In general, neural networks and deep learning are not extremely intuitive and they may be stuck at local optima or overfit. For more intuitive results in shorter time, we can use random forests - an ensemble method using multiple decision trees and returning the "mean" prediction at the end - and naive bayes.¶

Oversampled classes may lead classifiers to be biased in predictions of one class over another. Notably, there are many more negative tweets about airlines than other sentiments.¶

One way to handle majority classes is to perform the synthetic minority oversampling technique (SMOTE). An example is shown below from a paper I wrote.¶

def kFoldKerasModel(X,Y):
    for train, test in tqdm(kfold.split(X,Y)):
        XTrainSeq = tokenizer.texts_to_sequences(X.iloc[train])
        XTrainSeq = pad_sequences(XTrainSeq, maxlen=maxLength)
        XTestSeq = tokenizer.texts_to_sequences(X.iloc[test])
        XTestSeq = pad_sequences(XTestSeq, maxlen=maxLength)

Conclusions and Future Work¶

Based on the evaluations, the model that seems to perform best from the 10-fold cross-validation scores are the naive bayes, keras word embedding neural network and xgboost (gradient boosted trees). However, it appears that the estimate of overfitting being between validation and training errors from the cross-validation didn't reflect the rise in error within the test dataset for the keras neural network (0.4 accuracy from overfitting effects). For this reason, the keras neural network could undergo structural changes to reduce overfitting with dropout layers among other options.¶

For the test set, the pretrained vader classifier performed the best with the xgboost being second to it when comparing f1 score. The xgboost model has a relatively higher precision, which indicates less false positives. However, there seems to be more false negatives due to lower recall at 53.33%. One thing to note about benchmarks on pretrained models is that they are not being trained on the training set in the train/test splits in the 10-fold cross validation (CV). The main comparisons that make sense occur between the sklearn models that are trained on same 10-fold CV and evaluated.¶

A problem with the data itself is that some tweets don't have information about whether they're from the same conversation or what they're replying to or if they're starting a new conversation. We could be missing crucial information. One example is the first row of the airlineSentiment dataframe: "@VirginAmerica What @dhepburn said." The issue with this tweet is that it could be neutral, but also either of the other two sentiments. We're not sure what @dhepburn said, so we can't assess the sentiment of the sentence exactly.¶

For future work, we should focus on the data collection process that involves more extensive information on tweets such as a conversation number (so tweets in the same conversation are included). Also, we can look out for words that could be false positives - words that sound negative, but aren't.¶

For intuitive models, sklearn library models may suffice. For better accuracy, we can consider long-short term memory neural networks and transfer learning or using different word embeddings.¶

A popular area of research currently is sarcasm detection. Investigation into detecting sarcastic tendencies could be expanded on for potential improvement in out-of-sample error.¶
